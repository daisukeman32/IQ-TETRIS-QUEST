# 案件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
| --- | --- |
| **プロジェクト名** | クイズテトリス |
| **目的** | 定番のパズルゲーム「テトリス」に「クイズ」の要素を組み合わせることで、プレイヤーが楽しみながら知識を深めることができる教育的エンターテインメントソフトウェアを開発する。 |
| **背景** | ゲームの楽しさと学習を両立させたいというニーズに応える。また、幅広い年齢層が一緒に楽しめるように、難易度調整機能を充実させることで、家庭内でのコミュニケーションツールとしての役割も担う。 |

## 2. 機能要件

| ID | 機能名 | 詳細 |
| --- | --- | --- |
| **FR-01** | タイトル画面機能 | ゲーム開始前に、プレイヤーが設定を選択するためのタイトル画面を表示する。 |
| **FR-02** | レベル選択機能 | タイトル画面にて、以下の2つの軸で難易度を選択できる。<br>1. **対象者:** 「低学年」「高学年」「おとな」から選択。選択に応じてクイズの内容が変化する。<br>2. **スピード:** 「かんたん」「ふつう」「むずかしい」から選択。選択に応じてブロックの落下速度が変化する。 |
| **FR-03** | ゲームプレイ機能 | 1. 選択されたレベルに基づき、ブロックの落下とクイズの出題を開始する。<br>2. クイズに正解するまで、ブロックの操作は不可とする。<br>3. 正解後は、キーボードの矢印キー（左、右、下、上）でブロックを操作できる。<br>4. ブロックが着地すると、次のブロックとクイズが出現する。 |
| **FR-04** | スコアリング機能 | ブロックで横一列が揃うと、そのラインが消去され、スコアが10点加算される。 |
| **FR-05** | お助け機能 | 1ゲームにつき1回のみ使用可能。クイズに答えずに、そのターンのブロックを操作する権利を得る。 |
| **FR-06** | ゲームクリア機能 | スコアが規定値（100点）に達すると、「ゲームクリア」となり、特別な演出を表示してゲームを終了する。 |
| **FR-07** | ゲームオーバー機能 | ブロックが画面の最上段まで積み上がると、「ゲームオーバー」となり、ゲームを終了する。 |
| **FR-08** | サウンド機能 | 以下のタイミングでBGMおよび効果音を再生する。<br>・BGM: ゲームプレイ中<br>・効果音: UI操作、クイズの正誤、お助け機能使用、ゲームクリア時 |
| **FR-09** | タイトル復帰機能 | ゲームオーバー画面、またはゲームクリア画面から、クリック一つでタイトル画面に戻ることができる。 |

## 3. 非機能要件

| ID | 要件名 | 詳細 |
| --- | --- | --- |
| **NFR-01** | ユーザーインターフェース | 子供から大人まで、誰でも直感的に操作方法を理解できる、洗練された分かりやすいデザインとする。全体的に丸みと明るい色調を用いた、ポップで可愛らしい印象を与える。 |
| **NFR-02** | 実行環境 | 最新の主要なウェブブラウザ（Google Chrome, Firefox, Safariなど）で動作すること。OSに依存せず、特別なソフトウェアのインストールを必要としない。 |
| **NFR-03** | 拡張性 | 将来的にクイズの問題を追加・変更することが容易なデータ構造とすること。 |

## 4. 成果物

本プロジェクトによって作成される成果物は以下の通り。

1.  `index.html` - アプリケーションの骨格となるHTMLファイル
2.  `style.css` - アプリケーションの見た目を定義するCSSファイル
3.  `game.js` - アプリケーションのロジックを記述するJavaScriptファイル
4.  `README.md` - 開発者向けの簡易的な説明ファイル
5.  `案件定義書.md` - 本ドキュメント
